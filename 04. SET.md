# SETs (Combining Rows from Multiple Tables)
To combine to tables there should be the same columns 
#### Syntax
```sql
SELECT A, B
FROM C
JOIN X
WHERE <clause>
GROUP BY <column>
HAVING <claude>

UNION 

SELECT D, E
FROM F
JOIN X
WHERE <clause>
GROUP BY <column>
HAVING <claude>

ORDER BY <column>
```

### Rules
- Can use all clauses (WHERE, JOIN, GROUP, HAVING)
- Can use Order By only once
- Number of columns, Order of columns and their data types must be exact.
- Column Alias is defined in the first query

## UNION
- Removes duplicate and returns all distinct rows from both query
## UNION ALL
- Returns all rows including duplicates
- Generally faster than UNION
- Only SET Operator that returns duplicates
## EXCEPT
- Returns distinct rows from the first query that are not in the second query 
- Removes Duplicates from the results
- A row is considered the same only if every column in the select matches between the two results
```postgresql
SELECT FirstName, LastName
FROM Employees
EXCEPT 
SELECT FirstName, LastName
FROM Customers
```

## INTERSECT
- Returns only rows that are common in both queries 
- A row is considered the same only if every column in the select matches between the two results
```postgresql
SELECT FirstName, LastName
FROM Employees
INTERSECT 
SELECT FirstName, LastName
FROM Customers
```

- Combining similar information before analyzing data (Union and Union All)
- Identifying the difference (delta) or changes between two batches of data (Except)
