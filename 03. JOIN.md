# JOINs (Combining Columns from Multiple Tables)
- To combine two tables there should be a key column 
Used In: 
- Recombine Data (un-normalizing the data)
- Data Enrichment
- Check for existence 

## Inner
- Only the matching rows from both tables (Data from table A that has a match in table B)

```sql
SELECT * 
FROM A
INNER JOIN B 
-- ON <condition>
ON A.Key = B.Key
```

## Left/Right
- Return All Rows from Left/Right Table and only matching from Right/Left
- Get All Rows fromt he Primary table and addtional data from only matching rows in the secondary
- Will have NULLs when not matching in the secondary 
- Just use the LEFT JOIN and reverse the tables.

```sql
SELECT * 
FROM A
LEFT JOIN B 
ON A.Key = B.Key
```
## LEFT Anti Join
- Return Rows from LEFT that has no match in RIGHT
- No JOIN we get the effect by filtering key is null
- Get Customers who haven't placed any Order.

```sql
SELECT *
FROM A
JOIN B
ON A.Key = B.Key
WHERE B.Key IS NULL
```

## Full
- Return All rows From Both Tables
- Will have NULL when not matching 
- Order of tables in the query is not important 

```sql
SELECT * 
FROM A
FULL JOIN B 
ON A.Key = B.Key
```

## FULL Anti JOIN 
- Returns Only Rows that didn't match in either table
- Only Unmatching Data
- Customers that didn't make any orders and Orders that are not attached to a customer 

```sql
SELECT * 
FROM A 
FULL JOIN B 
ON A.Key = B.Key 
WHERE A.Key IS NULL OR B.Key IS NULL
```

## Cross JOIN 
- Combines Every Row From LEFT with Every Row From Right
- All Possible combinations (Cartesian Join)
- It gets crazy number of rows and make the database busy 

```sql
SELECT * FROM A CROSS JOIN B
```
