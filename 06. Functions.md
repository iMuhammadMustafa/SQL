# Single - Row Functions
One Value IN => One Value Out
## String 
#### Manipulation
- CONCAT(..args)
- UPPER(arg)
- LOWER(arg)
- TRIM(arg): Removes Leading/Trailing Spaces 
- REPLACE(char1, char2): Replaces a specific character with another
#### Calculation
- LEN
#### Extraction
- LEFT(Value, Nr of characters): Extracts specific number of characters from the start
- RIGHT(Value, Nr of characters): Extracts specific number of characters from the end
- Substring(Value, start, length): Extracts a part of the string at a position
	  - Substring (firstName, 3, 4)
	  - Substring (firstName, 2, LEN(firstName))
  ![[06.1 Functions - Substring.png]]
## Numeric
- Round(Num): Will round to a Num decimal numbers, it will either round up or round down depending on the Num+1 digit
- ABS(Num): Will convert a negative number to positive
## Date & Time

#### Part Extraction
**Part: month/mm, year, day, quarter, hour, week**
- DAY(date), MONTH(date), YEAR(date): Integer => Returns an Integer for the day/month/year
- EOMONTH(date): Date => returns the date and the day component is set to the last day of the month (2025-01-15 => 2025-01-31)
- DATEPART(part, date): Integer => Returns the part as Integer
- DATENAME(part, date): String => Returns the part as String (varchar)
- DATETRUNC(part, date): DateTime => Truncate the date to a specific part (minimal level of part we need and will reset what is after that)
###### Use Case
- Reporting and Aggregate date based on dates: 
	- How many orders were placed each year
- Data Filtering
	- How many orders were placed in January


#### Format & Casting
- FORMAT(value, format, culture?): string
- CAST(value AS newDataType) => Cast a value to a new type
- CONVERT(targetDataType, value, style?): => Change the value to a different type and formats it  (CAST and FORMAT)

#### Calculations
- DATEADD(part, value, column): => add a value to a part of column (adds 2 years to the year part of a date) DATEADD(year, 2, OrderDate)
- DATEDIFF(part, startDate, endDate) : => Find the difference between two dates

#### Validation
- ISDATE(value): 1/0 => returns 1 if the date is a valid date or 0 if it's not a valid date

## NULL
- ISNULL/IFNULL(value, replacement): Replaces the column NULL values with a replacement value (It could be another column)
	- ISNULL(ShippingAddress, 'Unknown')
	- ISNULL(ShippingAddress, BillingAddress)
- COALESCE(value1, value2, ..): Replaces the NULL value with the first non-null value 
- ISNULL/IFNULL is faster but limited to two value and not supported everywhere
- NULLIF(value, value2): If two values are equal it will return null otherwise it will return the first value 
	- Use Case to turn empty string and empty spaces to a NULL
	  NULLIF(TRIM(Category), '') => This will Trim Category and if it matches '' it will set the value to NULL

## Window 
Perform aggregation on a specific subset of data without losing th level of details of the row. Like a Group By.
![[06.3 Window.png|541]] 
![[06.4 Window Function.png]]
Frame Clause: 
- Frame Type : ROWS , RANGE
- Lower Frame Boundary: Current Row, N Preceding,  Unbounded Preceding
- Higher Frame Boundary: Current Row, N Following, Unbounded Following

- ROW_NUMBER(): Numbers the rows starting from 1 (1,2,3,4,...)
- RANK(): Numbers the rows starting from 1 but it gives the same number to equal values and it skips the next number (1,2,2,4,...)
- DENSE_RANK(): Similar to RANK but it will not skip ranking (1,2,2,3,4,...)
- NTILE(Number): Splits data into buckets (Bucket Size = Rows Count/Number)
	- Integer numbers, the larger groups come first then the smaller

- `CUME_DIST()`: calculates the distribution of data points within a window
	- Position Nr / Number of Rows
	- If duplicates, it will put the last position number for the same value on all of the duplicates (if duplicate they will share the same percentage)
	- inclusive (the current row is included)
- `PERCENT_RANK()`: calculate the relative position of each row within a window
	- Position Nr - 1 / Number of Rows -1
	- If duplicates, it will put the first position number (if duplicate they share the same percentage)
	- exclusive (the current row is excluded)

- LAG(exp, offset, default): Get previous row
	- Exp is the column to read
	- Offset from current row (default is 1)
		- Default value if column is null
- LEAD(exp, offset, default: Get next row
- FIRST_VALUE(exp): first row
- LAST_VALUE(exp): last row

```
SELECT
Result Current
LAG(Result,1,0) OVER (Partition By ProductId Order By Date DESC) Last
Current - Last / 100
FROM 
Sales;
```
