# Data Defination Language (DDL)

- CREATE

```sql
CREATE TABLE table_name (
    column1 datatype,
    column2 datatype,
    ...
)

CREATE TABLE Persons IF NOT EXISTS (
  Id INT,
  LastName VARCHAR(255),
  FirstName TEXT, -- VARCHAR without length is treated as TEXT in postgreSQL or VARCHAR(MAX) in SQL Server
  Age INT
)
```

- ALTER

```sql
ALTER TABLE Persons ADD COLUMN Address VARCHAR(255);
ALTER TABLE Persons DROP COLUMN Age;
ALTER TABLE Persons RENAME TO People;
ALTER TABLE Persons RENAME COLUMN LastName TO Surname;
ALTER TABLE Persons MODIFY COLUMN FirstName VARCHAR(100);
ALTER TABLE Persons ADD CONSTRAINT PK_Persons PRIMARY KEY (Id);
ALTER TABLE Persons ADD CONSTRAINT FK_Persons_Department FOREIGN KEY (DepartmentId) REFERENCES Departments(Id);
```

- DROP

```sql
DROP TABLE IF EXISTS table_name;
DROP TABLE IF EXISTS Persons;
DROP TABLE IF EXISTS People;
```

- TRUNCATE

```sql
TRUNCATE TABLE table_name;
TRUNCATE TABLE Persons;
```

- RENAME

```sql
RENAME TABLE old_table_name TO new_table_name;
```

- COMMENT

```sql
COMMENT ON TABLE table_name IS 'This is a comment';
COMMENT ON COLUMN table_name.column_name IS 'This is a comment';
```

- Difference between DROP and TRUNCATE:
  - TRUNCATE will remove the data from the table but will keep the table itself.
  - DROP will remove the table and all its data permanently.

# Data Manipulation Language (DML)

- INSERT

```sql
INSERT INTO table_name (column1, column2, column3, ...) VALUES (value1, value2, value3, ...);

INSERT INTO Persons (Id, LastName, FirstName, Age) VALUES (1, 'Smith', 'John', 30);

INSERT INTO Persons VALUES (2, 'Doe', 'Jane', 25); -- If you want to insert values for all columns, you can omit the column names
```

- UPDATE

```sql
UPDATE table_name
  SET column1 = value1,
      column2 = value2, ...
  WHERE condition;

UPDATE Persons
  SET Age = 31
  WHERE Id = 1;
```

- DELETE

```sql
DELETE FROM table_name WHERE condition;

DELETE FROM Persons WHERE Id = 2;
```

- MERGE
Combine data from one table into another by: 
* INSERT data that doesn't exist
* UPDATE data that does exist and they match
* DELETE data based on condition

```sql
MERGE INTO target_table AS t
USING source_table AS s
ON t.id = s.id

WHEN MATCHED THEN 
	UPDATE SET t.name = s.name
WHEN NOT MATCHED THEN
	INSERT (id, name) VALUES (s.id, s.name)
WHEN NOT MATCHED BY SOURCE THEN
	DELETE;
WHEN MATCHED AND s.status == 'VOID' THEN
	DELETE;

```

- SELECT INTO: 
Creates a new table based on the resut of a query

```sql
SELECT column1, column2, .. 
INTO new_table
FROM existing_table
WHERE Condition;
```

```sql
SELECT * FROM Employees
INTO Employees_Backup
WHERE status == 'active'
```

* SELECT INTO creates a new table and insert into it
* INSERT INTO inserts into an existing table
* Use Cases: Backup, temporary table to work on, saving result for later processing, data migration
* In Postgres the syntax is instead of INTO the query starts with `CREATE TABLE new_table AS` and then the select statment
